*git init

The staging area (intermediate place where you
can pick and choose which files inside your working directory
that you want to commit)

*git status - allows to view the staging are and see if files
	      are in it or not

to add files t the staging area and to start tracking changes
in it, we have to use the command:

*git add xxx (or git add . - to add everything in the working
	      directory

*git commit xxx (can add -m "xxx", to add a message)

*git log - shows what commits have been made
